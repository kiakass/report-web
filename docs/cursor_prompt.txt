웹페이지 구축

Title : 기업 고객 경쟁사 리포트

1. 목적
기업타겟인사이트 리포트는 기업부문의 데이터판매/제휴 및 영업 활동을 지원하기 위한 도구로, 
대상 기업의 
B2C 고객 가입 및 행동 데이터를 기반으로 인구통계학적 특성, 시계열 추이, 웹 방문 패턴, 고객 특성 지수 등을 주요 경쟁사와 비교 분석하여,
정량적 인사이트 및 차별화 포인트를 리포트 형식으로 제공을 위한 site 구축

2. 현재 단계
- 서비스 제공을 위한 Prompt page ( enterprise-report.html ) 을 만들었음.
- 메뉴구성 
  1) Main page : 내용소개
  2) 리포트생성  : 대상기업, 비교고객의 정보를 입력
  3) 히스트리    : 생성된 리포트의 히스토리를 목록으로 저장
  4) 설정       : 필요한 설정정보 저장

- 웹 서버 정보
  1) OS : linux - ubuntu
  2) IP : 122.36.63.100
  3) 웹서버 : Nginx
  4) 프론트페이지 : http://122.36.63.100/enterprise-report.html

프론트페이지를 MVP로 개발하여 웹서버에 올려놓은 상태임

3. 이후 진행
- 서비스컨셉 메뉴구성에 맞게 현재 구축된 페이지를 모듈화 해야함
- 각 메뉴에 기능을 추가 개발

# 메뉴구조
- 홈
- 리포트 생성하기
- 분석 히스토리
  ㄴ 분석 리포트

[홈] 페이지 기능

▶ 서비스 사용 시나리오  Title/소개, 좌측메뉴바, 주요기능소개

> 서비스 좌측메뉴바 클릭시 해당 페이지로 이동
> 주요기능 > 기능소개 (아이콘) 클릭시 해당 페이지로 이동
> 하단에 분석 서비스 현황 조회

[리포트 생성하기] 메뉴 기능

▶ 서비스 사용 시나리오  리포트정보/분석정보입력 > 리포트 생성하기
> 리포트생성 ‘클릭’ 시 페이지 로딩
> 분석 기간은 전월부터 1년이 최대로 조회됨
> 연령은 20~60세, 나머지 연령대는 제외
> 대상/비교 고객 : URL/고객명 입력 검색 -> 조회된 결과를 저장/삭제기능 필요

▶ 기능 개발 내역
1) 분석 설정 정보 입력 내용 저장
    ㄴ DB에 요청 내용 저장
    ㄴ DB, Table 설계
    ㄴ 입력정보 : 목적/설명/기업명/URL, 분석기간, 나이, 기업정보 상세(제품등)
2) 데이터 파일 업로드
    ㄴ 업로드 항목 ( 주제 :  파일 )
        ㄴ 인구통계학적분석 : 파일검색(로컬)
        ㄴ 시계열분석           : 파일검색(로컬)
        ㄴ 주요방문사이트    : 파일검색(로컬)
        ㄴ 교차방문율           : 파일검색(로컬)
    ㄴ 업로드가능파일종류 : xlsx, csv
    ㄴ local PC 에서 파일 선택하여 업로드
    ㄴ 서버에 파일을 저장하고 위치를 DB에 저장
    
3) 분석항목 선택 기능 (체크박스)
    ㄴ 분석항목 : 
        ㄴ 인구통계학적분석(MAU,연령별,성별,지역별,고객특성지수,가구특성별) : default check
        ㄴ 시계열분석 : 월별접속자추이, 월별점유율분석 : default uncheck
        ㄴ 고행동분석 :  주요방문사이트, 교차방문율, 성별/연령별 교차방문 분석 : default uncheck

4) 리포트 생성하기
    ㄴ 1), 2) 에서 생성된 정보를 로딩
    ㄴ 1), 2) 정보를 결합하여 선택된 3)분석항목에 대해 순차적으로 AI-agent 호출
        ㄴ 호출시는 지침(instrument:CR_INST.txt)의 내용을 참조하여 호출
            ㄴ 분석항목 - 요건 - 출력(Chart, 인사이트)
    ㄴ 순차적 호출/출력 항목에 대한 결과 저장
       ㄴ. Chart + 인사이트 : 지침 및 파일(report_form.html) 스타일/포멧 참조
	ㄴ. Chart 는  Chart.js 사용하여 작성/저장
            ㄴ. 인사이트는 입력된 정보 및 LLM 에서 생성한 내용 기반으로 답변
    ㄴ 웹기반 리포트 생성 정보 저장, 히스토리>상태 업데이트


[분석 히스토리] 메뉴 기능

▶ 서비스 사용 시나리오  리포트조회, 검색, 클릭 > 분석상세내용 페이지 이동

> 히스토리 메뉴진입시 내가 생성요청했던 리포트 확인 가능해야함
> 리포트 검색 가능해야함 : 리포트명, 번호, 생성일
> 리포트명/조회 클릭시 ‘경쟁사분석리포트’ 상세 내용 조회
> 번호/리포트명/고객명/분석기간/생성일/상태 내용이 조회됨

▶ 기능 개발 내역
1) ID 기반의 리포트 요청 내역 리스트 조회 기능
2) 리스트 검색기능
3) 리스트 정렬기능
4) 리스트 클릭시 상세페이지로 이동
5) 리스트 저장시 상세내용을 pdf 로 저장

[분석 히스토리]-[분석 리포트] 상세 페이지 기능

▶ 서비스 사용 시나리오  리포트조회, 검색, 클릭 > 분석상세내용 페이지 이동
> 클릭한 리포트 내역을 볼수 있다.

▶ 기능 개발 내역
1) 분석 리포트 진입시 개요 페이지 진입
2) 각 섹션별 메뉴 클릭시 상세 내역 조회 기능
